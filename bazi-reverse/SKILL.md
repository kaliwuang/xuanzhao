# 七术互逆 · 全入口交叉辩论系统

## 核心理念

七术中**任意一门**都可以是入口：

```
         ┌ 八字四柱 ──────────┐
         │ 紫微斗数 ──┐       │
         │ 奇门遁甲 ──┤       │
  输入 ──┤ 六爻纳甲 ──┼→ 倒推 → 排全七术 → 辩论
         │ 大六壬   ──┤   出生     │
         │ 太乙神数 ──┘            │
         └ 西洋占星 ──────────────┘
```

**一句话**：你给我七术中任一个盘，我还你全七术交叉辩论。

## Step 1：识别入口类型

用户输入 → AI 判断属于哪门术数的数据：

| 输入示例 | 判断为 | 依据 |
|---------|--------|------|
| 乙酉 壬午 甲子 庚午 | 八字四柱 | 四个干支组合，标准格式 |
| 命宫天相在午，财帛武曲天府 | 紫微斗数 | 十二宫+主星格局 |
| 阳遁三局，值符天冲星 | 奇门遁甲 | 局数+值符值使 |
| 本命卦火风鼎 | 六爻纳甲 | 卦名或六爻配置 |
| 四课寅申巳亥，三传申戌子 | 大六壬 | 四课三传术语 |
| 太乙积年XXX，主算XXX | 太乙神数 | 积年+主客算 |
| 太阳双子，上升处女，月亮狮子 | 西洋占星 | 星座+行星位置 |

## Step 2：各入口 → 倒推出生时间

### A. 从八字四柱倒推 ✅ 已实现（脚本）

```bash
python ~/.hermes/skills/xuanxue/bazi-to-seven-arts/scripts/reverse_bazi.py \
  -y 乙酉 -m 壬午 -d 甲子 -t 庚午 --json --brief
```

**原理**：日柱在60天周期内唯一，月柱限定节气范围，年柱锁定年份。脚本扫描候选年份逐日匹配。
**精度**：日柱+月柱+年柱锁定到天，时柱锁定到2小时窗口。

详见 `reverse_bazi.py`。

### B. 从紫微斗数倒推 ⚠ 技术可行

**关键数据提取**（需要用户提供以下任意组合）：

| 数据项 | 来源 | 对倒推的帮助 |
|--------|------|-------------|
| 命宫地支 | 命宫位置 | 月+时 的组合约束 |
| 紫微星所在地支 | 紫微星位置 | 日柱约束（紫微星由日干起） |
| 五行局 | 命宫干支 | 验证用 |
| 生年年干 | 四化起始 | 年柱约束 |
| 大概年龄 | 用户 | 解决60年周期歧义 |

**算法思路**（统一搜索框架）：

```python
# 伪代码：所有入口共用同一搜索框架
def find_birth_from_chart(constraints, arts_range=(1900, 2100)):
    """constraints: dict of partial chart data from any art"""
    for y in candidate_years:
        for m in 1..12:
            for d in 1..days_in_month:
                for h in possible_hours:
                    # 生成全七术数据
                    charts = generate_all_seven(y, m, d, h)
                    # 检查与输入是否匹配
                    if match_constraints(charts, constraints):
                        candidates.append((y,m,d,h))
    return candidates
```

**从紫微倒推详细步骤**：
1. 用户提供命宫地支 + 紫微星位置（如"命宫在午，紫微在寅"）
2. 命宫在午 → （月+时）组合限定：命宫从寅起数，数到午 = 第5个位置
   - 命宫 = (月支序号 + 时支序号 - 2) % 12? 不对...
   - 命宫公式：命宫 = (14 - 月支序号 - 时支序号) % 12，其中寅=2... 
   - 不对，让我查一下标准公式。
3. 紫微星位置 → 紫微星口诀："紫微"由日柱的 日干+日支 决定，有特定查表
   - 这可以反推日柱的范围

实际上，紫微倒推比八字倒推复杂得多，因为紫微星的位置取决于日柱的 天干+地支，映射关系复杂。

对于紫微入口，最佳方式是：
1. 让用户提供尽量多的数据（命宫、紫微星位置、五行局、大概年份）
2. 使用统一搜索框架，在候选日期范围内生成紫微盘，匹配用户提供的盘面特征
3. 匹配命宫 + 紫微星位置 通常能锁定到2-3个候选日期

### C. 从西洋占星倒推 ⚠ 技术可行

**关键数据**：
- 太阳星座 → 月份（±15天）
- 上升星座 → 出生时间（±2小时）
- 月亮星座 → 农历月日约束
- 其他行星位置 → 进一步缩小

**算法**：
```
太阳星座锁定月份 → 上升星座锁定时辰 → 月亮星座锁定具体日期范围
 → 在范围内验证行星位置 → 输出候选日期
```

**工具**：`pyswisseph` 可以计算任意日期的星盘，反向扫描即可。

### D. 从奇门遁甲倒推 ⚠ 理论可行

**关键数据**：局数（如"阳遁三局"）、值符天星、值使门

**原理**：
- 局数 → 节气约束（阳遁/阴遁 + 局号 → 节气范围，约15天）
- 值符天星 + 值使门 → 日柱 + 时柱约束
- 组合后可锁定到1-3天

### E. 从六爻纳甲倒推 ❌ 精度低

本命卦由日柱决定，同一日柱每60天出现一次。
缺少时柱信息时精度严重不足，仅能作为辅助约束。

**最佳策略**：要求用户补一年份或月份，或提供更多卦象特征。

### F. 从大六壬倒推 ⚠ 技术可行（精度有限）

**关键数据**：四课三传的天将组合、课式类型（贼克/比用等）

**原理**：
- 天将由日干决定（如甲日青龙在寅）
- 课式由日柱 + 时辰决定
- 配合年份范围可缩小候选

### G. 从太乙神数倒推 ⚠ 理论可行

**关键数据**：太乙积年、主算/客算、太乙临宫

**原理**：
- 太乙积年有固定公式：某年 + 固定偏移
- 主客算由日时决定
- 可反向推演

## Step 3：排全七术

拿到出生时间后，一律全排七术：

- 八字 → `lunar_python`
- 紫微 → `iztro-py`
- 奇门 → `kinqimen`
- 六爻 → `najia`
- 六壬 → `kinliuren`
- 太乙 → `kintaiyi`
- 占星 → `pyswisseph`

各库的具体调用细节见对应 skill，但以下实测API签名需注意：
- **kinqimen**：字典key用**繁体中文**（'離'/'兌'/'乾'/'巽'，非'离'/'兑'/'干'/'巺'），用简体字查询会返回空
- **kinliuren**：`from kinliuren.kinliuren import Liuren`，构造 `Liuren(year, month, day, hour)`
- **najia**：`NaJia(year=2005, month=6, day=9, hour=11, minute=50)`，注意是关键字参数
- **kintaiyi**：`TaiYi(year=2005, month=6, day=9, hour=11)`
- **pyswisseph**：星历文件路径需 `swe.set_ephe_path()` 设置

**执行策略**：排六术时使用 `delegate_task` 并行启动子任务（最多3路并行），每路子任务负责一门术数的排盘+判词撰写。前三术排完后启动后三术。

本 skill 只负责编排和辩论的编译，不负责各术的盘面计算细节。

## 统一搜索框架（底层逻辑）

所有入口共用同一搜索算法：

```python
def universal_reverse(constraints, year_range=(1900, 2100)):
    """
    通用倒推：从任意术数的部分盘面信息，反推出生时间。
    
    constraints = {
        'art': 'ziwei' | 'qimen' | 'liuyao' | 'daliuren' | 'taiyi' | 'astrology' | 'bazi',
        'data': {...},  # 各术特定的盘面数据
        'known_age': None,  # 可选：已知年龄用于缩小范围
    }
    """
    for y in candidate_years:
        for m in months:
            for d in days:
                for h in hours:
                    charts = compute_all_seven(y, m, d, h)
                    if verify(charts, constraints):
                        yield (y, m, d, h)
```

不同入口的区别在于 `verify()` 函数的匹配逻辑 + `candidate_years` 的优化策略。
八字最精确（日柱唯一），紫微次之（星曜组合），其余精度递减。

## 工作流（AI 执行顺序）

```
1. 接收用户输入
2. 判断入口类型（八字/紫微/奇门/六爻/六壬/太乙/占星/原始生日）
3. 若为原始生日 → 直接排七术
4. 若为任一门术数 → 运行对应倒推逻辑
   a. 八字入口 → reverse_bazi.py
   b. 紫微入口 → 统一搜索框架（命宫+紫微星约束）
   c. 占星入口 → 统一搜索框架（太阳+上升+月亮约束）
   d. 其他入口 → 统一搜索框架 + 用户补充约束
5. 如有多个候选，让用户确认年龄段或提供更多特征
6. 确认出生时间后，排全七术 → 用 delegate_task 并行启动6路子任务
   - 批次1（3路并行）：紫微 · 奇门 · 六爻
   - 批次2（3路并行）：六壬 · 太乙 · 占星
   每路子任务独立排盘、撰写判词。
7. **第一步输出：七术盘面全展示**
   - 排好七术后，**先并列展示七个板块的盘面数据**
   - 八字用表格格式，紫微列十二宫主星，奇门列九宫+值符值使，六爻列六亲，六壬列四课三传，太乙列积年主客算，占星列行星+上升
   - ⚠ 只展示数据，不加分析。用户要求：\"先展示盘面，再开始分析\"
   - 展示完后加分隔线 `━━━━━` 再进入辩论
8. **第二步输出：三碟辩论**
   - 各子任务返回判词后，按三碟格式编译
   - 八字判词由主任务自己撰写（已知数据最多）
   - 入口术的判词可略详（50-100字）
   - 三碟：各有判词 → 交叉印证 → 综合判定
9. **可选第三步：衔接玄照96视角辩论**
   - 用户说\"108个人辩论\"或七术后表示"不够"时运行
   - 详见"Step 5：衔接玄照96视角辩论"
```

**致命陷阱**：有用户明确纠正过\"只展示七术盘面不做辩论\"的问题。必须先展示盘面，再输出辩论。七术全展示→再分析，缺一不可。

**注意**：delegate_task 最多3路并行（max_concurrent_children=3），六术分两批次执行。每路子任务给的 context 必须包含出生时间和性别。

## Step 5：衔接玄照96视角辩论（用户期望的第二步）

七术辩论是"**专家小组**"（7位中国传统术数专家）。
玄照96视角是"**群体智慧**"（96位跨学科智者）。

**用户期望**：给出七术辩论后，用户可能紧接着要求96视角群体辩论。
**默认行为**：七术辩论和96视角辩论是**互补而非替代**，都应当提供。用户说"108个人辩论"时，实际运行的是96个视角。

### 触发方式

用户说"108个人辩论"、"群体智能"、"玄照"等，或七术辩论输出后用户表示"不够"时：

```bash
cd ~/tmp/xuanzhao
python3 -c "
from perspectives_engine import DeepPerspectiveEngine
from xuanzhao import DestinyAnalyzer

engine = DeepPerspectiveEngine()
analyzer = DestinyAnalyzer()
destiny = analyzer.analyze('出生日期 时间', '地点', '性别')

all_ids = list(engine._frameworks.keys())
results = engine.analyze(destiny, all_ids)

# 分析评分分布，找出共识与分歧
scores = [(pid, r.get('score',0), r.get('confidence',0)) for pid, r in results.items()]
scores.sort(key=lambda x: -x[1])
# 然后基于评分数据编译辩论输出
"
```

### ⚠️ 玄照关键陷阱

1. **实际视角数96，非108**：`perspectives_engine.py` 有20个深度视角类（Perspective子类）+ 76个模板视角，共计96
2. **默认只输出12个**：xuanzhao.py 第824行 `deep_count = 12`，第828行 `[:deep_count]` 切片。`analyze` 命令不支持 `--deep` 参数，`report`/`predict` 命令支持
3. **全量视角约2分钟**：96个视角全跑约60-120秒
4. **kinqimen 数据字典key用繁体中文**：'離'/'兌'/'乾'/'巽'而非'离'/'兑'，用简体查询会返回空
5. **kinliuren 导入路径**：`from kinliuren.kinliuren import Liuren`

### 辩论序列规则

| 顺序 | 辩论类型 | 内容 | 时间 |
|------|---------|------|------|
| 先行 | 七术辩论 | 7位术数专家各自判词+交叉印证 | ~2分钟 |
| 随后 | 108视角辩论 | 96位跨学科智者群体共识+分歧 | ~2分钟 |

用户可能只要求其中之一，也可能要求两者。默认输出七术后询问是否继续108视角。

## 边界与注意事项

- **入口信息越完整，倒推越精确**：命宫+紫微星+生年四化 比单纯命宫好很多
- **60年周期歧义是所有入口的共同问题**：都需要用户提供年龄段或大概年份
- **入口术的判词可以更详细**：用户给了某术的盘，自然对该术最熟悉
- **如果倒推精度太低**（3个以上候选），输出七术辩论时加上"日期存疑，基于最近候选"的声明
- **辩论格式统一**：不论从哪门术入口，辩论格式完全一致
